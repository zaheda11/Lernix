FAQ & Guide

Frequently Asked Questions regarding the LERNIX curriculum framework.

What is LERNIX?

LERNIX is a highly advanced AI-powered Curriculum Planning platform that transforms course titles into industry-aligned semester courses. Unlike basic syllabus lists, it maps course intelligence (difficulty, workloads, career paths), writes comprehensive learning outcomes, and dynamically plots prerequisite pathways.

How is the curriculum output generated?

The backend queries the IBM Granite 3.3 2B model running locally via Ollama. By utilizing custom JSON-schema prompts, we force the LLM to output valid educational outlines, mapping previous courses to current semester prerequisites, preventing academic hallucinations.

What if I don't have Ollama running locally?

No worries! The application contains a built-in Mock Generator Fallback. If the connection to your local Ollama port (11434) fails, LERNIX automatically supplies dynamic, industry-standard curriculum designs for Computer Science, Data Science, and Design courses, making the platform fully functional out-of-the-box.

How do I configure my local Ollama setup?

Follow these simple steps:

  1. Download Ollama from ollama.com and install it.
  2. Open your shell and run the command: ollama run granite3.3:latest or the model configuration name set in llm_service.py.
  3. Restart the Flask server. It will automatically detect the local container on port 11434!
What roles are supported on the platform?

LERNIX has three distinct user roles:

  • Student: Can view all roadmaps, study plans, dashboards, generate personalized study guides, and review Q&A preps. Students must opt into curricula first.
  • Educator: Can generate curricula, edit structures, review student checklist completions, and export PDFs/JSONs.
  • Admin: Accesses overall database logs, role distributions, user feedback files, and detailed performance scores.
